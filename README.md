# Крестики-нолики (Tic-Tac-Toe)

## 1. Описание игры

- **Правила игры**  
Игроки по очереди ставят `X` и `0` на поле 3×3. Побеждает тот, кто первым выстроит три своих символа в ряд (по горизонтали, вертикали или диагонали). Если все 9 клеток заполнены, а победителя нет - ничья.

- **Логика победы/поражения**  
Победа определяется сразу после хода. Проигрыша как такового нет - либо победа, либо ничья, либо игра продолжается.

- **Особенности реализации**  
Игра полностью на JavaScript без использования сторонних библиотек. Состояние хранится в массиве `board`. Интерфейс обновляется через DOM. Есть кнопка перезапуска.

## 2. Игровая механика

- **Как определяется победитель?**  
Функция `checkResult()` перебирает все выигрышные комбинации из массива `winConditions`. Если три одинаковых непустых символа совпадают - объявляется победитель.

- **Как обрабатываются клики?**  
На каждую ячейку (`cell`) повешен обработчик `handleCellClick`. Он получает `data-index` ячейки, проверяет, свободна ли она и активна ли игра. Затем обновляет `board`, выводит символ в ячейку и вызывает `checkResult()`.

- **Как работает перезапуск?**  
Кнопка `restart` вызывает `restartGame()`, которая сбрасывает `currentPlayer` на `X`, очищает `board` (заполняет пустыми строками), устанавливает `gameActive = true`, обновляет текст статуса и очищает текст всех ячеек.

## 3. Структура данных

- **Массив `board`**  
Хранит состояние игрового поля: 9 строк, каждая - `"X"`, `"0"` или `""` (пусто). Индексы от 0 до 8 соответствуют ячейкам.

- **Условия победы**  
Массив `winConditions` содержит 8 комбинаций индексов, которые образуют линию (3 горизонтали, 3 вертикали, 2 диагонали). Пример: `[0,1,2]` - верхний ряд.

- **Как хранится состояние игры?**  
Переменная `gameActive` (boolean) - активна ли игра. `currentPlayer` - кто ходит сейчас. Пока `gameActive = true`, ходы разрешены.

## 4. Алгоритмы

- **Алгоритм проверки победы**  
  ```js
  for (let i = 0; i < winConditions.length; i++) {
    const [a, b, c] = winConditions[i];
    if (board[a] && board[a] === board[b] && board[a] === board[c]) {
      roundWon = true;
      break;
    }
  }
  ```
  Проверяет каждую комбинацию: если все три клетки не пусты и равны - победа.

- **Алгоритм определения ничьи**
После проверки победы, если `roundWon === false` и в массиве `board` нет пустых строк (`!board.includes("")`) - ничья.

- **Алгоритм смены игрока**
Если нет победы и ничьи, то `currentPlayer = currentPlayer === "X" ? "0" : "X"`. Затем обновляется текст статуса.
